# Haryel Ramalho's Skills

A public collection of **130 agent skills** for local development workflows - **5 personal** skills, **22 curated** skills, and **103 community** skills. Each skill provides domain-specific instructions, procedures, reference material, scripts, templates, or checklists for a focused task.

## Installation

### Quick install (recommended)

Install this repository into your skills directory with the [`skills`](https://www.npmjs.com/package/skills) CLI:

```bash
npx skills add https://github.com/haryelramalho/skills
```

### Install a single bucket

Use the `owner/repo/<subpath>` shorthand to install only one bucket:

```bash
# Only personal skills
npx skills add haryelramalho/skills/skills/mine

# Only curated skills
npx skills add haryelramalho/skills/skills/curated

# Only community skills
npx skills add haryelramalho/skills/skills/community
```

You can also pin a specific skill with `--skill`:

```bash
npx skills add haryelramalho/skills/skills/mine --skill multi-tool-branch-review
```

### Manual install

Copy or symlink the skills you need into your local skills directory:

```bash
# Copy a single skill
cp -r skills/mine/multi-tool-branch-review ~/.agents/skills/multi-tool-branch-review

# Or symlink an entire bucket
ln -s $(pwd)/skills/mine ~/.agents/skills/mine
```

Skills are organized into top-level buckets:

- `skills/mine/` - 5 personal skills maintained here
- `skills/curated/` - 22 selected skills promoted for easier discovery
- `skills/community/` - 103 community skills

## Usage

Skills are picked up when placed in your configured skills directory. Each skill declares its trigger behavior and workflow in `SKILL.md`.

## What are Skills?

Skills are structured instruction sets for repeatable workflows. Each skill lives in its own directory under `skills/` and contains a `SKILL.md` file with metadata, procedures, and optional reference material. Skills follow the [agentskills.io](https://agentskills.io) specification.

## Skill Catalog

## Structure

Each skill follows a consistent directory layout:

```text
skills/
  mine/<skill-name>/       # Personal skills
  curated/<skill-name>/    # Selected skills
  community/<skill-name>/  # Community skills

skills/<bucket>/<skill-name>/
  SKILL.md       # Main skill definition (required)
  references/    # Deep-dive reference material
  examples/      # Usage examples and patterns
  templates/     # Templates and scaffolds
  scripts/       # Automation scripts and validators
  checklists/    # Step-by-step verification checklists
```

## Contributing

To add a new skill:

1. Create a directory under the appropriate bucket with a lowercase, hyphenated name:
   - `skills/mine/` - personal skills
   - `skills/curated/` - selected skills promoted for discovery
   - `skills/community/` - community skills
2. Add a `SKILL.md` with proper frontmatter, including a `description` field.
3. Include reference material, examples, templates, scripts, or checklists when needed.
4. Keep this README catalog in sync with the directory structure.
